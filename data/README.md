# Data Card

## Dataset

`data/raw/articles.csv` is generated by `src/generate_data.py`.

Columns:

- `doc_id`: stable document identifier
- `date`: article date
- `title`: generated headline
- `text`: generated article body
- `labels`: JSON list of labels from `fraud`, `money_laundering`, `sanctions_violation`, `none`
- `source`: generation source, usually `synthetic_template`

## Generation Method

The generator creates short news-style articles using randomized templates, entities, dates, countries, dollar amounts, and investigation language. The default target distribution is approximately:

- 55% `none`
- 45% adverse-media documents split across `fraud`, `money_laundering`, and `sanctions_violation`
- 10-15% of adverse documents with overlapping adverse labels

Negatives include clean business, financing, operations, and regulatory-news articles that mention companies, people, and locations without alleging wrongdoing.

## External Text

The generator tries to import a small set of real neutral headlines/text snippets through HuggingFace `datasets` when `--include-real-negatives` is used and dependencies/network access are available. If that fails, generation continues with synthetic-only text and records this limitation in the run summary printed by the script.

## Limitations

- Synthetic text can overrepresent wording patterns.
- Entity names are fictional.
- Labels are generated from templates, not human annotation.
- This is a portfolio/demo dataset and should not be used for production AML, sanctions screening, SAR filing, or customer-risk decisions.
